# 2026-05-04 — Dashboard v2 Phase 05 / Step 01 — Rework reason enum

## What shipped

**Replaces fragile `strings.Contains(reason, "test"/"lint"/...)` bucketing in `qa_queries.go::ReworkCauses` with a typed `RunOutcomeReason` enum.** The QA Rework widget now groups runs by canonical reason keys instead of free-text substring matches.

### Backend

- `internal/db/run_outcome_reason.go` — `RunOutcomeReason` type + 9 constants (test_fail, lint_fail, human_reject, timeout, policy_violation, error, user_interrupted, agent_finished, other) + `ClassifyRunOutcome(runID)` helper that prefers `events` signals (test.fail, lint.fail, policy.violation, approval.denied, run.timeout) over the wrapper's free-text `runs.session_end_reason`.
- `internal/db/v8_to_v9_migration.go` — Go-side data rewrite. Handles **both legacy JSON shapes**: `map[string]int` (production wrapper output) and `[]map[string]any` (Phase 04 ad-hoc seed). Idempotent via `schema_version` gate. Malformed JSON rows preserved verbatim.
- `internal/db/qa_queries.go` — `ReworkCauses()` body shrinks from 35 lines to 25, no `strings` import needed. Returns 9 buckets in canonical order, padded with zeros. Unknown enum keys fold to `other`.
- `internal/attribution/compute.go` — `aggregateSessionStats` now calls `d.ClassifyRunOutcome(s.RunID)` instead of bumping `agg.Outcomes[s.SessionEndReason]`. New runs get classified at attribution time.
- `internal/db/schema.go` — `SchemaVersion = 9`.
- `internal/db/migrate.go` — v8→v9 branch added.

### Frontend

- `cmd/web/dashboard/app.js::renderQARework` — palette upgraded to a 9-color map keyed by canonical reason name (was 5-color positional array). Filters out zero-count buckets so the donut isn't dominated by empty categories.
- `cmd/web/dashboard/index.html` — widget subtitle updated from hardcoded 5-bucket list to "Canonical RunOutcomeReason buckets (9, fixed order)".

### Demo seed

- `internal/demo/seed.go::SeedCrossProject` — adds 3 `task_attribution` rows (one per project), each with 4-bucket `session_outcomes` JSON keyed by enum strings. Live verification shows the donut populated with test_fail=6, lint_fail=3, human_reject=3, timeout=3 across the cross-project demo.
- `ResetDB` now also wipes `task_attribution` for clean reseeds.

## Notes / debt left

- Wrapper itself (`internal/wrapper/`) still writes only 3 values to `runs.session_end_reason` (`agent_finished`/`user_interrupted`/`error`). Finer reasons (`test_fail`/`lint_fail`/`timeout`/...) only surface when matching events exist on the run. Acceptable for now — Layer-3 events are populated when the run goes through the lint/test wrapper. Pre-Phase-05 historical runs without those events stay in 3-bucket world even after migration.
- `LegacyBucketReworkReason` lives in `run_outcome_reason.go` (used by migration only). It's the only string-match logic left in the codebase and should be deleted in a future cycle once we're confident no v8 DBs remain in the wild.
