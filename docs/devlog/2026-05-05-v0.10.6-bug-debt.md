## Drift from plan

- Bug 3 used `db.Begin()` instead of `BEGIN IMMEDIATE`. The driver (`modernc.org/sqlite` with `MaxOpenConns=1`) already serializes at the connection level, so explicit BEGIN IMMEDIATE would require a non-standard isolation string with `BeginTx`. Standard `Begin()` gives the same correctness here. Noted in code comment.
- Bug 11 needed slightly more than "delete one line" — `pIdx` was a loop variable elsewhere becoming unused, so the `for pIdx, p := range` was changed to `for _, p := range`. Same intent, slightly larger diff.
- Line numbers in plan vs current code drifted (codebase grew); fixes applied at correct logical locations.

## Compliance items deferred (documented as limitations)

The plan listed 3 compliance/security items that aren't bugs but needed decisions. All deferred to documentation rather than fixed:

1. **No CSRF on `POST /api/audit-log/verify`** — read-only endpoint. Dashboard binds to loopback since v0.10.5, so blast radius is minimal. Documented in CHANGELOG known-limitations.
2. **Anchor model protects only rows ≤ `last_audit_id`** at anchor time — newer rows unpinned until next anchor. Documented.
3. **Confluence upsert best-effort under network errors** — if `UpdatePage` fails after `SearchPages` succeeded, no local anchor recorded. Documented.

## Lessons

- Bundling deferred bugs into a single patch release worked well — half-day of focused work, one commit, zero coordination overhead. Better than dripping fixes one-per-PR.
- Code review report → plan → implementation pipeline scaled cleanly: severity-ranked table in the report became the implementation order in the plan, and the plan's exact bug numbering became the commit body.
- The two added tests gave confidence on the two MED bugs (race + upsert) — the LOW bugs are obvious-correct.
