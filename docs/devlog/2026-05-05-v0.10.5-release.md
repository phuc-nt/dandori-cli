## Release-readiness fixes (this commit)

Code-reviewer flagged 2 blockers. Both fixed inline:

### HIGH — Dashboard LAN bind

`cmd/dashboard.go` previously bound to `:port` (all interfaces). On any shared network this exposed engineer names, Jira keys, cost data, and the full audit log to anyone on the LAN, with **no authentication layer**.

Fix: default `--bind` to `127.0.0.1`; opt-in `--bind 0.0.0.0` prints a warning. 9 lines.

### MED — Audit verifier false-positive on DB errors

`VerifyAuditChainWithAnchors` previously treated every `row.Scan` error as "row missing → tamper detected". A locked DB, dropped connection, or driver error therefore raised a false `anchor mismatch` — eroding trust in the headline anchor feature.

Fix: distinguish `sql.ErrNoRows` (real tamper, keep current message) from other errors (return the error to caller). 5 lines + 2 imports.

### Bonus — gofmt sweep

6 files had stale formatting (`internal/db/{eng,po,qa}_queries.go`, `run_outcome_reason.go`, `v10_to_v11_migration.go`, `internal/server/po_endpoints.go`). Ran `gofmt -w`.
