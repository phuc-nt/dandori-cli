# 2026-05-04 — v0.10.0 Dashboard v2 (Phase 01)

## What shipped

**Dashboard v2 foundation** — UI rebuild around the persona-tab architecture. Phase 01 of the v0.10.0 plan covers the cross-cutting infrastructure (filter bar, alert center, KPI strip, DORA traffic light, run drawer, mobile responsive, demo data) that every persona tab will consume in Phase 02–04.

9 steps shipped, each gated by markup tests + headless live snapshot:

- Step 1: Embed.FS extraction (`cmd/web/dashboard/{index.html,style.css,app.js}`)
- Step 2: Hash-router + persona sidebar (Overview / DORA + G9 / Agents / Runs / Costs / Quality KPI)
- Step 3: Unified Alert Center (bell + dropdown panel, persisted ack via `alerts_acked` table)
- Step 4: Sticky control bar (header + filter pills pinned during scroll)
- Step 5: KPI strip — 4 tiles + inline SVG sparklines + WoW delta with semantic coloring (lowerIsBetter inversion for cost/tokens)
- Step 6: DORA Scorecard collapsed to single traffic-light row (`🟢🟢🔴🟢 DORA: 2 elite, 1 high, 1 low · click to expand`) → click opens detail modal with the existing 4-tile grid
- Step 7: Run Detail drawer (slide-in from right, 5 tabs Summary/Events/Quality/Files Changed/Audit-disabled, ESC + backdrop close, deep-link `?run=<id>`)
- Step 8: Mobile responsive 375 / 768 / 1440 — sidebar collapses < 768px, KPI strip stacks 1-col < 480px, filter pills wrap, run drawer goes full-width
- Step 9: Cross-project demo seed (`dandori demo --seed --variant cross-project` → 3 projects × 3 sprints × 4 runs = 36 runs)

## Schema migration

Schema v7 → v8: added `alerts_acked` table + 3 composite indexes (`runs(jira_sprint_id, started_at)`, `runs(department, started_at)`, `runs(git_remote, started_at)`). Migration is idempotent — `runDashboard` now calls `Migrate()` so v7 DBs upgrade transparently on first dashboard launch.
