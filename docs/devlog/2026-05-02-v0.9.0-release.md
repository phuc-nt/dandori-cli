- Phase 05: Quiet flag (global `-q`, mutual exclusion with `-v`)
- Phase 06: Watch daemon orchestration (launchd/systemd-user, `enable/disable/status`)

**Single afternoon build + test pass + docs.** Then a docs sweep confirmed all stale references (symlink workaround, alias shell sourcing) were gone.

## Lessons learned

1. **Init wizard complexity** — 13 prompts is long, but users only see it once + upfront test connection feedback paid dividends (nobody asks "why is my Jira failing" post-init anymore).

2. **Alias footgun removal was bold** — breaking change, but the alternative (supporting both shell alias + explicit subcommand) created surface area. Clean break + migration docs was simpler long-term.

3. **Verify gate flip to OFF** — Phase 04 was the riskiest. Explicit `true` configs are honored (no silent behavior change), but new users get OFF by default. This reduced "every task run fails verification" support burden.

4. **Docs stale reference sweep** — grep found 0 matches for `alias claude` and `/tmp symlink` post-sweep. Audit trail shows 16 refs removed from 2 docs during Phase 07 (docs). Automation (CI lint) would have caught this earlier.
