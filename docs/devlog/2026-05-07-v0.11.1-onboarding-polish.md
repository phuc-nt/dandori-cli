## Why this patch

The UX audit graded v0.11.0 a B+: "Strong foundation. Needs UX in feedback loops and guided discovery to earn an A." Three friction points stood out:

1. **Silent run success** â€” solo user types `dandori run -- claude "..."`, it exits with no signal. Confidence drops on first try.
2. **Hidden v0.11 features** â€” affinity matrix, RCA, trend chart shipped but discoverable only via `--help`. Init wizard's "Next steps" still pointed at `analytics all`.
3. **Confluence default = Yes** â€” solo path defaulted to enabling a team-only integration, then got stuck on the space-key prompt.

All three are 5â€“20 lines of code; no architecture, no data model. The patch fits in one session.

## Drift from spec

- Spec said `--quiet` should skip only the "View:" line. Implemented as: the entire summary block is gated by `--quiet`. Reason: simpler mental model, and `--quiet` users almost certainly want full silence. No behavioral footgun.
- Doctor's "skipped" state uses a neutral `-` glyph rather than a warning glyph â€” fits the "not a failure, just not configured" intent.
- A legacy `checkConfluence(ok bool)` helper remains alongside the new `checkConfluenceResult(skipped bool)` â€” kept for compatibility, removable in v0.12 cleanup.

## Lessons

- UX audits between releases are cheap and high-leverage. The walkthrough that surfaced these 3 fixes took ~30 minutes of focused exploration and yielded a release-worthy patch.
- "Silent success" is a recurring trap for CLI tools â€” every long-running command benefits from a 1-line acknowledgment.
- Defaults shape user mental models more than explanatory text. Flipping Confluence to `[y/N]` does more than any wizard reword.
